Transana's Text Reports

Transana provides a variety of reports that allow you to explore the Quotes, Clips, Snapshots, and Notes you've created and the coding you've applied to your data using Keywords.

Library Report

The Library Report produces a list of Documents and Episodes, including Keywords in the selected Library. This allows you to review the default Keywords you have assigned to all Documents and Episodes in a Library. To generate a Library Report, right-click the name of a Library in the database tree and choose Library Report from the pop-up menu.

This report can be customized in a variety of ways. The Filter Dialog allows you to customize the report content and specify what Documents, Episodes, and Keywords should be included or excluded in the report. You can edit this report directly, save it as a rich text format or HTML document, and print it. When you have finished looking at the report, click Close to exit.

Document Report

The Document Report produces a list of all Quotes created from the selected Document, and can optionally include Quote text, Quote position information, Quote Keywords, Quote comments, and Quote Notes. To generate a Document Report, right-click the name of a Document in the database tree and choose Document Report from the pop-up menu.

This report can be customized in a variety of ways. The Filter Dialog allows you to specify what data elements should be shown, and what Quotes and Keywords should be included or excluded in the report. You can edit this report directly, save it as a rich text format or HTML document, and print it.

When configured appropriately, the Document Report includes a brief summary of the coding that has been applied to the items listed in the report.

When you have finished looking at the report, click Close to exit. Please note that the Document Report is structurally similar but analytically very different from the Collection Report.

Episode Report

The Episode Report produces a list of all Clips created from the selected episode and the Snapshots associated with the Episode. The Episode Report can optionally include Clip Transcripts, Clip Keywords, Clip comments, Clip Notes, and Snapshot Notes. To generate an Episode Report, right-click the name of an Episode in the database tree and choose Episode Report from the pop-up menu.

This report can be customized in a variety of ways. The Filter Dialog allows you to specify what data elements should be shown, and what Clips, Snapshots, and Keywords should be included or excluded in the report. You can edit this report directly, save it as a rich text format or HTML document, and print it.

When configured appropriately, the Episode Report includes a brief summary of the coding that has been applied to the items listed in the report. This summary reports two different measures of coding frequency, item count and media time covered by those items.

When you have finished looking at the report, click Close to exit. Please note that the Episode Report is structurally similar but analytically very different from the Collection Report.

Collection Report

The Collection Report produces a list of all Quote, Clips, and Snapshots gathered together in the specified Collection, and can optionally include Quote text, Clip Transcripts, item Keywords, comments, and notes related to included Collections, Quote, Clips, and Snapshots. To generate a Collection Report, right-click the name of a Collection in the database tree and choose Collection Report from the pop-up menu.

This report can be customized in a variety of ways. The Filter Dialog allows you to specify what data elements should be shown, and what Quotes, Clips, Snapshots, and Keywords should be included or excluded in the report. By default, Quotes, Clips, and Snapshots in nested Collections are included in the report, though this can be changed through the Filter dialog. You can edit this report directly, save it as a rich text format or HTML document, and print it. When you have finished looking at the report, click Close to exit. Please note that this is structurally similar but analytically very different from the Document and Episode Reports.

Notes Report

The Document, Episode, and Collection Reports can be configured to show Collection, Quote, Clip, and Snapshot Notes for items included within the different scopes of those reports. This can be very useful when looking at Notes related to particular subsets of your data.

You can also look at your Notes across your entire database. The Notes Reports are available from within the Notes Browser tool. You can produce reports for Notes associated with Library records, Document records, Episode records, Transcript records, Collection records, Quote records, Clip records, Snapshot records, or all Notes in a database. The Filter Dialog allows you to specify which Notes are included in the report.
