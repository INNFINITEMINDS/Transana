The File Management Tool

The File Management tool is designed primarily for moving and organizing video and audio files. It allows users to manipulate files on a local network, as well as transferring files between a local network and the Storage Resource Broker, a repository useful for sharing media files with distant collaborators.

The File Management screen is divided into two sides with a set of controls between them. Files can be copied or moved in either direction.

Copy will copy selected files from the current directory on one side of the screen to the current directory on the other side of the screen in the direction of the arrow button pressed. Please note that the File Management Tool will NOT over-write existing files, and will skip the copy if the file already exists in the target directory. If you want to replace a file, you must delete it before copying the new file.

Move will move selected files from the current directory on one side of the screen to the current directory on the other side of the screen in the direction of the arrow button pressed. Please note that the File Management Tool will NOT over-write existing files, and will skip the move if the file already exists in the target directory. If you want to replace a file, you must delete it before moving the new file.

Copy All New will copy all files from the current directory and all subdirectories on one side of the screen to the current directory on the other side of the screen, creating subdirectories as needed, in the direction of the arrow button pressed. The File Management Tool will NOT copy a file if that file already exists in the target directory, even if the source version is newer. If you want to replace a file, you must delete it before requesting Copy All New.

Delete will delete all selected files from the current directory on the indicated side. Directories can be deleted as well, but only if they are empty.

New Folder will create a new directory on the indicated side.

Update DB will update the location of media files in Episode and Clip definitions in the Transana database. The File Management Tool will look up all references to the selected media files on the indicated side and will update the path reference, taking the Video Root setting into account, for those database records. This function is extremely useful if you change the location of a media file in relation to your defined Video Root setting. Only file locations on the local network can be used for updating a database. It is not possible to use a file stored only on the Storage Resource Broker with Transana due to bandwidth issues.

Connect is used to connect to the Storage Resource Broker, or SRB, and to disconnect from it. The SRB provides secure, encrypted file transfer over the internet. If your project needs access to a Storage Resource Broker for transferring files, please use the Contact form on the Transana Web Site to request a user name and password for your project. As a general rule, do not change the default values on the SRB Connection form unless you know what you are doing.

Refresh will update all directory and file information displayed by the File Management tool to reflect changes that may have been made by other processes interacting with these resources.

Close will close the File Management tool.

You may notice that the File Management Utility does not provide a way to rename media files. To preserve data integrity related to millisecond accuracy of start and stop positions, we have not provided a way to change the media file associated with Clips, other than changing the PATH (but not the File Name) provided by the Update DB button. We strongly recommend that you do NOT change file names of media files once you have begun working with them in Transana. If you do, you will have to recreate your Clips.
