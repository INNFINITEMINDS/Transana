Search

To search for a single keyword, you can right-click a keyword in the data window's database tree and select Quick Search from the popup menu. A node named after the keyword group : keyword pair will appear under the Search node containing a tree structure with all episode, clip, and snapshot records that have the selected keyword.

If you want to do a more complex search, right-click on the Search root node on the database tab of the data window and select Search to search your Episodes, Clips, and Snapshots for the presence of assigned keywords. The Search dialog will appear.

The Search dialog box has been carefully designed to only allow valid entries and to produce valid searches. If something you want to choose is not available, it is because you have missed something required to make your specification complete. For example, if the "Search" button is not enabled, it is most likely because you have not closed a set of parentheses or because you just selected a logical operator and have not yet specified the next Keyword Group : Keyword pair.

When you have constructed a valid search query, you can save it for later reuse by pressing the  icon in the lower left corner of the dialog. You can then load the saved search later using the  icon or delete it using the  icon.

This dialog box allows you to specify complex searches, including using boolean logical operator (AND, OR, and NOT,) but you must do so carefully to be sure to get the results you want.

For example, the searches "(A or B) and C" and "A or (B and C)" produce different results. The first search will find all Episodes, Clips, and Snapshots that have Keyword C and also have either Keyword A or Keyword B, producing hits on combinations AC, BC, and ABC. The second search will find all Episodes, Clips, and Snapshots that have Keyword A, as well as those that have both B and C, producing hits on A, AB, AC, BC, and ABC. These are obviously very different results, with the second search including more results.

This is particularly an issue with the more ambiguous search "A or B and C." Many people initially expect this to be equivalent to the first query above. However (because AND is higher than OR in the Boolean "order of operations"), this search is actually equivalent to the second search described above, and will find all Episodes and Clips that have Keyword A, as well as those that have both B and C. To avoid problems, liberal use of parentheses is recommended.
