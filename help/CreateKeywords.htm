Creating Keyword Groups and Keywords

Overview

As you probably know, coding is a central activity in many forms of qualitative analysis. While it would be unusual to start a qualitative analysis with a well-developed coding scheme, it is not at all unusual to want to identify source files with some of their primary characteristics from the beginning of the analytic process. There are often some initial codes that apply to an entire source data file that a researcher wants to make sure get applied throughout analysis. In education research, the name of the teacher, the school, the classroom subject or grade level, and the curriculum being used are common examples.

This is where we will begin working with Transana. The Demonstration data consists of three files; a text document describing the analysis of text in Transana, a video file describing the analysis of media in Transana, and a photograph that lends itself to simple qualitative analysis. During the Tutorial analysis, you will create a small database that documents the initial stages of the analytic process within Transana to which you can refer later should you need a refresher.

Keywords

Let's start by creating some Keywords that describe the different data item types one encounters in Transana.

In Transana, you use the Data window to create and manage Keywords in order to develop a coding system.

Activity: Follow these directions (Macintosh single-button mouse users, remember that Control-click emulates right-click):

1. To create a Keyword Group, right-click the Keywords node in the database tree, then click Add Keyword Group.

You see the following dialog box:

2. Enter Tutorial as the Keyword Group name and click OK. Transana creates your first Keyword Group.

(Please note that this Keyword Group will not be saved in the Transana database until you put some Keywords in it! If you quit Transana right now, that group will not exist when you restart the program.)

3. The next step is to add a Keyword to this group. Right-click the Tutorial Keyword Group, then click Add Keyword from the resulting menu. Type in the Keyword "Document". Under the optional definition, we might include a description that says that a Document is a source text file that has been brought into Transana for analysis. We will ignore the other fields for the moment. When done, click OK to save the Keyword.

4. Now add the keyword Episode, defined as representing one or more overlapping media files brought into Transana for analysis. Add another for Library, described as an organizational node for grouping source Documents and Episodes, Transcript, defined as a written representation of the contents of an Episode, and Keyword, defined as a tag used for coding analytic data within Transana. Note that each keyword is added to the database tree as you enter it.

5. Now right-click the Keywords node in the Database Tree and select the Keyword Summary Report option from the menu. You will see a listing of the Keywords you have created, organized by Keyword Group, including your Keyword definitions.

This report can be a very useful as a coding dictionary for your project. You can produce a similar report for a specific Keyword Group by right-clicking the Keyword Group and selecting the Keyword Summary Report option.
