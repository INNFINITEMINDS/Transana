Searching for Keywords

Transana lets you search for clips and episodes that contain specified keywords. Searches are named, and, once created, they appear in reverse chronological order under the Search node in the database tree. This lets you keep track of what you've searched for and what you found when you searched. You can save the results of a search as a collection, but search results are not otherwise permanent.

Quick Search

A Quick Search in Transana is one designed to show all clips that have been coded with a single keyword.

Activity: Follow these directions to perform a quick search:

1. Right-click the keyword Symmetry in the Tutorial keyword group in the database tree in the Data window. Choose Quick Search from the pop-up menu.

Transana creates a Search Result named Symmetry under the Search node. The program finds six clips that have the keyword Symmetry assigned to them. Subsequent quick searches for the same keyword will be numbered.

Notice that the series/episode and collection/clip structure you have set up is preserved in the search results. This search returns a small set of results, so it's not that helpful, but when you have a very large, complex data set, this helps you know which clips have been returned in the search results.

Standard Search

Standard searches allow you to do more complex searches, combining multiple keywords using boolean logic.

Activity: Perform a standard search using the keywords Symmetry and Confusion. Follow these directions:

1. Right-click the Search node in the database tree in the Data window and choose Search from the pop-up menu. Transana opens the Boolean Keyword Search dialog box

2. Type Symmetry and Confusion in the field labeled Search Name.

3. Click Symmetry in the list of keywords in the dialog box.

4. Click Add Keywords to Query.

5. Click the AND button under the label Operators in the dialog box.

6. Double-click Confusion in the list of keywords in the dialog box to add it to your search.

7. Notice that the Search button is now enabled. Transana's search dialog is designed to only allow legal searches. If the Search button is not enabled for you, it's because you have open parentheses or a dangling boolean operator or some other mistake in your search specification.

8. Notice in the lower left hand corner of the search dialog box that there are buttons that allow you to load, save, and delete search specifications. While we will not use these now, they can be useful when you wish to design a search you will use repeatedly as your data evolves.

9. Click Search to begin searching.

Transana displays the results of your search in the database tree.

This search produced four clips, which you can review by right-clicking the Symmetry and Confusion Search Result node and selecting Play All Clips. In one clip, a student suggests that symmetry may have something to do with houses. In the second, a student talks about the carpet being symmetrical if it's all gray. In the third, the class discusses symmetry as meaning things that are the same, and in the fourth, the teacher is asking about how students will know if they can always create symmetrical two-by-two designs.

The clip Proof in this instance is a false positive result. In this clip, the students are clear about the concept of symmetry, but are expressing confusion about the broader implication of their initial finding that they can always create symmetrical two-by-two designs. Let's drop it from your Search Result (but not from its original Collection) by right-clicking the clip in the search result and selecting Drop from Search Result from the pop-up menu.

Advanced Topic: The boolean operators used in Transana's search are subject to an "order of operations" just like other mathematical operators are. This has important implications for your searches which you need to understand if you will ever use more than two search terms. See Search in the Transana Manual for more information.
