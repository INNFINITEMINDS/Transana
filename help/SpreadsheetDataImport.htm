Import Spreadsheet Data

Please note that the Student version of Transana does not support importing Spreadsheet data. To use this feature, you must be using the Professionsal, Computer Lab, or Multi-User version of Transana.

If you have spreadsheet or survey data, you may want to import Comma Separated Values or Tab Delimited Text Files into Transana for analysis. Import Spreadsheet Data allows you to create a library of Documents based on the contents of a single data file in Comma Separated Values (*.csv) or Tab Delimited Text (*.txt) format. Transana provides a structured tool for such imports.

Such data is organized in a grid. You will find that your headings, prompts, or questions fall along one axis and your participants fall along the other axis. Such files can contain a variety of data types, including numeric, categorical, and text data.

These files are often in Microsoft Excel (*.xlsx, *.xls), Comma Separated Values (*.csv), and Tab Delimited Text (*.txt) formats. If your data is in Excel format or that of other spreadsheet programs, you will need to load your data in your Spreadsheet program and convert it to Comma Separated Values format (*.csv) or Tab Delimited Text format (*.txt) for import into Transana. More information on exporting data from Excel and about file encoding is available at the bottom of this page.

To begin importing a Spreadsheet Data file, right-click the Library where Documents should be created and choose "Import Spreadsheet Data".

On the first page of the Spreadsheet Data import process, you select the file you wish to import. Once you have selected a file, press the "Next >" button.

The next page shows the data in the first column of your data file and the data in the first row of your data file. One of these two data lists should show the headings, prompts, or questions in your data file, while the other should show the participant data associated with the first heading, prompt, or question. Use the checkboxes at the bottom of the form to let Transana know whether your headings, prompts, or questions appear in columns or rows in your data file.

The third page in the import process defines how to organize the data during import. There are three items to complete on this page:

At the top, you should identify a data item which uniquely identifies your participants. Data from this item will be used to indicate which participant is associated with that participant's responses. Researchers often use pseudonyms or ID numbers rather than using actual participants real names. If there is no unique identifier field that you want to include in your imported data, you can request that Transana create a unique identifier for each of your partiicpants.

In the middle of the page, you will see a list of all of the headings, prompts, or questions in your data file. You should select the data items you want to include in the Documents created during data import. Click in the list to select a single item. Use shift-Click to select a range of items after an initial selection. Use Ctrl-click on Windows or Cmd-click on OS X to select or de-select individual items from this list.

Finally, you should indicate whether you want your data imported into Documents for Participants or Documents for individual headings, prompts, or questions. If you select Participants, a Document will be created in your Library for each participant, using the unique identifier you selected above as the Document ID and including the prompt and response data for all items selected above. If you select Questions, a Document will be created in your Library for each heading, prompt, or question in your data file as the Document ID and including the unique identifier you selected and that participant's response to that question. All of the selected data is included in the Documents either way, but that data is organized in a fundamentally different way.

If you choose to organize your data by Participants, you will see an additional page that allows you to select what data items you want to use for automatic coding of your Documents. (Automatic coding is not possible when data is organized by questions.) Note that Keywords will be created in the "Auto-code" Keyword Group and will be applied at the Document level so that they will be presented as default Keywords when you create Quotes from the Documents. Transana imports the data ready to be coded. It does not automatically create coded Quotes from that data.

Exporting Data from Excel

Please note that this description is based on Excel 2010 for Windows. It is meant as a description of what works for me at this time, and you may need to change certain details and specific settings to get it working correctly for you.

If you are working in English or a language based on the Latin alphabet, including many western-European languages, you can use Excel to export a Comma Separated Values (*.csv) file. Load your spreadsheet in Excel. Choose File > Save As. On the Save As form, change the "Save as Type" to "CSV (Comma delimited) (*.csv)" and enter a file name. Press Save. This will save a plain text file with comma separated values using "cp1252" encoding. You will need to know that encoding value for importing the file into Transana.

If you are working on a language based on a non-Latin alphabet, such as Chinese, Japanese, Korean, Arabic, or Russian, Excel's CSV format will not support the characters in your language. Instead, you need to save a Tab Delimited Text file using a Unicode encoding. Load your spreadsheet in Excel. Choose File > Save As. On the Save As form, change the "Save as Type" to "Unicode Text (*.txt)" and enter a file name. (Please note that this is different than the "Text (Tab delimited) (*.txt)" format option.) Press Save. This will save a Unicode text file with tab delimited values using "UTF-16" encoding. You will need to know that encoding value for importing the file into Transana.

File Encodings

The English alphabet, made up of 26 upper case characters and 26 lower case characters and not including accents, is known as the ASCII character set. These 52 characters, along with numbers, punctuation symbols common to English, and a selection of special characters, are well known to computers and are handled the same way almost everywhere.

If your data includes characters that are not used in English, such as accented characters or non-Latin letters, there are a variety of different systems for storing characters not included in the ASCII character set, such as accented characters and non-Latin characters. Such systems for storing characters are known as encodings, and the general rule is that you need to know what file encoding was used to save a file in order to read and correctly interpret that file. Unfortunately, there is no commonly-accepted standard for specifying what encoding was used within a plain text file. You are just supposed to know what encoding was used.

If you are using Excel, please see the section on exporting data from Excel above. If you are using a different program, you will need to export your data into a Comma Separated Values file or a Tab Delimited Text file. You may be able to select an encoding when exporting a file. (If so, UTF-8 is usually a good choice.) If you are not able to specify what encoding to use, you will need to do a little research on the Internet to find out what encoding system your program uses by default. Either way, you will need to know the file encoding used by the program that saves the file in order for Transana to read and correctly interpret that file.
