Episode Properties

An Episode in Transana is a data object that links one or more media files with one or more Transcripts. Episodes hold a pointer to the media file(s) selected when they are created, so media files should not be renamed after being added to Transana. If you move your media files, you can use the File Management tool to update file locations in the database.

To add an Episode in Transana, right-click (Ctrl-click on the Mac) a named Library in the Data window and choose "Add Episode." You will see the dialog box shown above. The only required information is an "Episode ID" which must be unique within that Library and a "Media Filename" that identifies the video or audio file you wish to work with. "Date" should identify the date the media file was recorded, if known, and is especially helpful when working with large collections of media files. "Comment" can be used in any way you wish. The "Library ID" and the "Length" field are filled in automatically by Transana and cannot be edited.

By selecting a Keyword Group and Keyword, then clicking the  button, you can add as many analytic keywords as you wish to each Episode. These Keywords are default codes, and are optionally applied to all Clips created from the Episode. Selecting a Keyword Group : Keyword combination in the Keywords list and pressing  removes a keyword from the list, and pressing  allows you to access Keyword Management.

Please note that all Media Filenames must be unique at the file name level. That is, once you have created an Episode from a media file with a given name, do not use the same media file name to represent a different piece of video or audio media for analysis in Transana, even if that media file is stored in a different folder on your computer.

Note that Transana is able to display up to four media files simultaneously, if your computer is powerful enough. See Working with Multiple Media Files for more information on this feature.
